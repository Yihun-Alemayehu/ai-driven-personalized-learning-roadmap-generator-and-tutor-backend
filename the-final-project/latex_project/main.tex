\chapter*{Abstract}
\addcontentsline{toc}{chapter}{Abstract}

The proliferation of online educational content has created unprecedented opportunities for self-directed learning, yet it has simultaneously introduced a critical problem: the fragmentation of high-quality resources and the absence of structured guidance. In the Ethiopian context, where access to premium mentorship and formal course subscriptions is economically prohibitive, students often find themselves trapped in \enquote{tutorial hell} --- scrolling through disconnected YouTube videos, blogs, and documentation without a coherent learning path or performance feedback mechanism. Existing solutions (e.g., roadmap.sh) provide static, one-size-fits-all curricula that fail to adapt to individual learning pace, gaps, or learning preferences.

This project presents \projectname, a comprehensive web and mobile platform designed to synthesize free, high-quality technical resources into a dynamically adaptive, assessment-driven learning roadmap. The core innovation lies in a hybrid artificial intelligence architecture that mitigates hallucination risks inherent in pure Large Language Model (LLM) generation by employing a \enquote{Skeleton \& Flesh} strategy. Offline, a powerful LLM generates a verified, semantically coherent ontology (the learning path skeleton) covering tech domains (Frontend, Backend, Data Science, DevOps). At runtime, a lightweight tutor model generates context-specific explanations and quizzes (the flesh) tailored to each learner's performance and progression.

\projectname\ introduces three novel contributions: (1) the \textit{Gatekeeper Pattern}, an assessment-driven mechanism requiring learners to achieve \textgreater80\% on a three-question quiz before advancing with adaptive resource swapping if performance drops below 50\%; (2) \textit{Mastery Decay Integration}, incorporating Ebbinghaus's Forgetting Curve to remind learners of concepts not reviewed for 14+ days, promoting retention through spaced repetition; and (3) \textit{Multi-Path Branching}, enabling learners to select personalized learning trajectories based on preferences (visual vs.\ logical, speed vs.\ depth).

The platform integrates the Google Programmable Search Engine API to aggregate only verified resources from trusted domains (FreeCodeCamp, GeeksForGeeks, MDN, Coursera), ensuring content quality and accuracy. The technical stack comprises React and Flutter for responsive web and mobile frontends, Node.js for API orchestration, Python for ontology generation and AI logic, and PostgreSQL for persistent storage. Development follows Agile methodologies with continuous integration/continuous deployment (CI/CD) pipelines to ensure reliability and rapid iteration.

This project addresses a critical gap in educational technology for developing economies, democratizing access to structured, mentorship-quality guidance while remaining zero-cost to end users. By combining rigorous ontology-based curriculum design with intelligent adaptive assessment, \projectname\ empowers Ethiopian students and learners across resource-constrained regions to navigate the complex landscape of technical skill acquisition with confidence and clarity.

\noindent\textbf{Keywords:} adaptive learning, ontology-based curriculum design, artificial intelligence, assessment-driven progression, educational technology, spaced repetition, personalized learning paths, developing economy context.

\section{Core Algorithm Pseudocode}
\label{app:pseudocode}

The following pseudocode describes the core gatekeeper classification algorithm that drives assessment-driven progression:

\begin{lstlisting}[language=JavaScript, caption=Gatekeeper score classification]
function classifyScore(scorePercent):
  if scorePercent >= 80: return 'strong_pass'
  if scorePercent >= 70: return 'marginal_pass'
  if scorePercent >= 50: return 'fail_low'
  if scorePercent >= 30: return 'fail_fundamental'
  return 'fail_severe'
\end{lstlisting}

\section{Adaptive Difficulty Computation}
\label{app:difficulty}

The adaptive difficulty function calibrates quiz difficulty based on learner performance:

\begin{lstlisting}[language=JavaScript, caption=Adaptive difficulty calibration]
function computeAdaptiveDifficulty(bestScore, averageScore, nodeDifficulty):
  performanceDelta = (bestScore - averageScore) / 100
  adjustedDifficulty = nodeDifficulty + performanceDelta
  return clamp(adjustedDifficulty, 1, 5)
\end{lstlisting}

\section{Mastery Decay Transition Logic}
\label{app:decay}

The decay algorithm determines when a mastered node transitions to review or relearn state:

\begin{lstlisting}[language=JavaScript, caption=Mastery decay computation]
function computeDecayState(lastReviewedAt, masteredAt, isMarginalPass):
  daysSinceReview = now() - max(lastReviewedAt, masteredAt)
  threshold = isMarginalPass ? 7 : 14
  if daysSinceReview > 30: return 'relearn'
  if daysSinceReview > threshold: return 'review_needed'
  return 'mastered'
\end{lstlisting}
